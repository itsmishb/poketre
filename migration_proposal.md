# システム移行提案書：Google Apps Script から Google Cloud Run へ

## 1. 現状の課題とエラーの原因

現在、Google Apps Script (GAS) で運用している画像処理システムにおいて、以下の問題が発生しています。

- **タイムアウトエラー (`Exceeded maximum execution time`)**: 
    - **原因**: GAS には「1回の実行につき最大6分」という厳格な制約があります。一度に大量の画像（例：44枚以上）を処理しようとすると、Gemini API の呼び出しやファイル操作の合計時間が 6分を超えてしまい、処理が途中で強制終了されます。
    - **影響**: 未処理の画像が残る、あるいは処理済みかどうかの判別が不安定になる原因となります。
- **ロックエラー (`Server error occurred`)**: 
    - **原因**: GAS の `LockService` は Google のサーバー負荷が高い時に一時的なエラーを返すことがあります。
    - **影響**: 多重起動防止が正しく機能せず、スクリプトが起動に失敗します。

**結論**: GAS は少量の自動化には適していますが、**1日 100〜3,000枚といった大量の画像処理には不向き**な環境です。

---

## 2. 提案内容：Google Cloud Run への移行

GAS の代わりに、Google Cloud 上で動作する **Cloud Run** を利用する構成を提案します。

### メリット・デメリット比較

| 項目 | Google Apps Script (現在) | Google Cloud Run (提案) |
| :--- | :--- | :--- |
| **実行時間制限** | **最大 6分**（致命的） | **最大 60分**（大量処理も余裕） |
| **処理速度** | 遅い（シングルプロセス） | **速い**（並列処理が可能） |
| **安定性** | 稀にサーバーエラーが発生 | 非常に高い（商用レベル） |
| **運用形態** | ブラウザ上で直接編集 | プログラムとしてデプロイ |

## 3. コスト試算（月額目安）

最新の **Gemini 3 Flash Preview** を使用した場合の試算です。無料枠が従来の 1,500枚から **1,000枚/日** に変更されている点にご注意ください。

| 処理枚数 (1日) | 月間合計 | Gemini API 代 | Cloud Run 実行代 | 合計コスト (月額) |
| :--- | :--- | :--- | :--- | :--- |
| **100枚** | 約3,000枚 | 0円 (無料枠内) | 0円 (無料枠内) | **0円** |
| **1,000枚** | 約30,000枚 | 0円 (無料枠内) | 0円 〜 約200円 | **約200円程度** |
| **3,000枚** | 約90,000枚 | 約25,000円 〜 | 約500円 〜 | **約25,500円程度** |

> [!IMPORTANT]
> - **Gemini 3 Flash Preview のコスト**: 無料枠（1,000枚/日）を超えた分は従量課金となります。解析精度は非常に高いですが、1.5 Flash に比べるとトークン単価が高いため、3,000枚/日 などの大規模処理ではコストが上昇します。
> - **コスト優先の場合**: 1.5 Flash モデルを選択すれば、3,000枚/日の場合でも合計 **月額 3,000円程度** に抑えることが可能です。
> - **Cloud Run**: 月間 200万リクエストまで無料のため、課金は計算リソース使用分のみとなります。

## 4. セルフレビューと補足事項

本提案内容について、技術的な観点から以下の通りセルフレビューを実施しました。

- **モデルの選択**: Gemini 3 Flash Preview は最新の高性能モデルですが、コスト面では 1.5 Flash が圧倒的に有利です。用途に応じてモデルを切り替えられるよう、環境変数で設定可能な設計としています。
- **実現可能性**: Python による Google Drive/Sheets の操作は標準的なライブラリで完結しており、移行の確実性は非常に高いです。
- **制約事項の考慮**: 
    - Gemini 無料枠（1,000枚/日）には「1分間に15リクエストまで」という制限があります。プログラム側で待機(Sleep)を入れることでエラーを回避します。
    - **プライバシー**: 無料枠での利用時、入力データはモデル改善に利用される可能性があります。
- **スケーラビリティ**: Cloud Run は最大 60分まで連続実行可能なため、3,000枚をまとめて処理しても GAS のような中断の心配はありません。

## 5. 移行に必要な工数の見積もり

既に Python のメインプログラムや設定ファイル（Dockerfile等）の作成は完了しているため、残りの作業は主に **「Google Cloud の環境構築とデプロイ」** になります。

| 作業工程 | 推定工数 | 内容 |
| :--- | :--- | :--- |
| **GCP 環境セットアップ** | 約 1 時間 | プロジェクト作成、API有効化、CLIのログイン |
| **デプロイ・環境設定** | 約 1 時間 | プログラムのアップロード、APIキー等の環境変数設定 |
| **権限・共有設定** | 約 0.5 時間 | サービスアカウントへのDrive/Sheet共有 |
| **動作確認・微調整** | 約 1〜2 時間 | テスト実行、ログ確認、エラー時の修正 |
| **合計** | **約 4〜5 時間 (半日程度)** | |

> [!TIP]
> Google Cloud の操作に慣れている場合や、チャットでのサポートを受けながら進める場合、**最短 2〜3 時間** での実稼働も可能です。

---

## 6. 移行に必要な準備

1. **Google Cloud プロジェクトの作成**: [Google Cloud Console](https://console.cloud.google.com/) から作成。
2. **お支払い情報の登録**: 無料枠利用でも登録自体は必須です。
3. **API Key の発行**: Google AI Studio 等から発行。
4. **権限設定**: Cloud Run のサービスアカウントに対して、利用する Google Drive フォルダやスプレッドシートへの共有設定を行います。

---

## 7. まとめ

Google Cloud Run への移行により、GAS の時間制限を気にせず、かつ最新の Gemini 3 モデル等の恩恵を最大限に受けられるようになります。コストと精度のバランスを見ながら、最適なモデル運用をご提案いたします。
